< TCC Schedulling System >

TCC Scheduling System is a web application that give an interface with users to deliver service such as faculty schedule, admin monitoring panel, adding and removing subject, room, class schedule and other services that can be observed in scheduling system. For the OJT requirements, I collaborated on this project alongside my coworkers Chad Terrible and Jayson Garcia. This project makes use of PHP, Ajax, HTML, CSS, and Bootstrap. The goal of this project is to automate the manual process of TCC scheduling, allowing users to reduce their working time and other critical processes during scheduling. It also includes filtering for the entire data structure and to store all the data gathered by the system we use MySQL database.

Languages/tools:HTML,CSS, PHP, JScript, Ajax, BootStrap, MySql, Github, Visual Studio

The content below offers an overview of the source code used to develop the TCC Scheduling System project. The source code is subdivided into several sections: Frontend that include the CSS type, Backend that include the CRUD validation with MySql database, login.php, index.php, home.php, manage_schedule.php and admin_class.php for the Ajax data transfering with MySql Database. Becuase of the multiple file for the system functionality I decide to redirect the user for the viewing of overall files in this project. Kindly check the file here TCC Scheduling System Files.

Code Overview

login.php

This PHP source code contains the system's login function. It collects input data from system users and transfers it to the validation process to decide if the user is able to login or not. For the general functionality, this source code is a blend of style, php, JScript, MySql, and other computer languages or text scripts.

index.php

The Bootstrap modal connection method was added in this PHP source code for each form that has a modal in the front-end or client side. This code's functionality includes declaring the alert value of each input in the modal area during the PHP validation procedure. This code is widely used in the backend CRUD process using PHP by returning a value from server side.

home.php

This PHP source code contained the system's homepage after login. In this code, the system's functionality is selected, such as viewing the daily or monthly calendar using Bootstrap and JScript, linking buttons to different Bootstrap modals with their assigned services, and regulating the general operation of the system.

manage_schedule.php

This PHP source code includes the system's scheduling procedure, in which the system accepts data from the user in the case of adding their schedule. It happens utilizing Bootstrap modal through the index.php connection, and it includes the system's adding and updating procedure in terms of scheduling in this function. The admin class.php handles all data validation during the validation process.

admin_class.php

This is PHP source code for validation of data from the user to database and also during the client side data request.

view_faculty.php

This is the PHP source code for viewing the registered faculty and their class schedule information.

subjects.php

This is the PHP source code include the system CRUD process in course subject panel.
